get /partner/activations
List of Activations (getPartnerActivations)
Returns a list of activations for which the user can make API requests. There is a limit to the number of activations which can be returned at one time. If the first request does not return a full list, use the 'next' attribute in the response body to request the next page. By default, activations are returned in batches of 100. The maximum page size is 1000. Provide only valid filters. During filter the activations empty and invalid filter fields are ignored. In case the installer ID doesn't exist, API will return an empty array.

Query parameters

next (optional)
Query Parameter — If the first request does not return a full list, use the 'next' attribute in the response body to request the next page. By default, activations are returned in batches of 100. The maximum page size is 1000. If the returned list below the limit, then response does not include the 'next' field. default: null
limit (optional)
Query Parameter — There is a limit to the number of activations which can be returned at one time. default: null
stage (optional)
Query Parameter — Filter activations by stage. Passing in_progress alone will consider as you have passed all the 1,2,3,4 stages. Passing multiple stage values using comma to filter. E.g. stage=1,2,3. Passing in_progress with any other combination will give you empty systems. default: null
reference (optional)
Query Parameter — Filter activations by company reference. default: null
installer_id (optional)
Query Parameter — Filter activations by installer ID. default: null
system_name (optional)
Query Parameter — Filter activations by system name. default: null
address (optional)
Query Parameter — Filter activations by address. This will perform a partial search and return if match found with street1, street2, country, state, zipcode of the activation. default: null
region (optional)
Query Parameter — Filter activations by region. Provide two latitude, longitude values to find activations between them. Format 'latitude1,longitude1,latitude2,longitude2' e.g value '44.968046,-94.420307,44.33328,-89.132008'. default: null
search (optional)
Query Parameter — Filter activations by search. This will perform a partial search and return if match found with system_name, reference, system_id, street1 and street2, country, state, zipcode of the activation. default: null

Return type

GetPartnerActivationsResponse
